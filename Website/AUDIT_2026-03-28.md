## 4. Critical Issues (Fix Immediately)

### 4.1 `sitemap.xml` is malformed

**File**: `public/sitemap.xml`
**Severity**: HIGH (SEO)
**Impact**: Google Search Console will reject this sitemap, preventing proper indexing.

Current content (broken):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
 url>                                              <!-- BUG: missing '<' -->
    <loc>https://www.omniprise.com.py</loc>
    <lastmod>2026-03-28T16:57:39+09:00</lastmod>
</urlset>                                          <!-- BUG: missing </url> -->
```

**Fix**: Replace with valid XML:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.omniprise.com.py</loc>
    <lastmod>2026-03-28</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>
```

### 4.2 ESLint configuration is broken

**File**: `../eslint.config.mjs` (lives in parent directory)
**Severity**: HIGH (developer experience, code quality gate)
**Impact**: `npm run lint` fails. `next build` skips linting. No automated code quality checks.

```
Cannot find package '@eslint/eslintrc' imported from
/home/bruno-rivas/corporate-food-dashboard/eslint.config.mjs
```

**Root cause**: The ESLint flat config is in the monorepo root but `@eslint/eslintrc` is not installed there.

**Fix options** (pick one):
- **Option A (recommended)**: Move `eslint.config.mjs` into `Website/` and install `@eslint/eslintrc` as a devDependency in `Website/package.json`.
- **Option B**: Install `@eslint/eslintrc` in the root `package.json` if there is one.
- **Option C**: Migrate to the new ESLint flat config format without `FlatCompat` and remove the `@eslint/eslintrc` dependency entirely.

### 4.3 Production is behind local

The dark logo fix (`06a846c`) has not been pushed to GitHub. Vercel deploys from GitHub, so production does not have this fix.

**Fix**: `git push origin main`

### 4.4 No favicon

**Severity**: HIGH (branding)
**Impact**: Browser tab shows no icon. Looks unprofessional.

No `favicon.ico`, no `apple-touch-icon.png`, no `site.webmanifest` in `public/`.

**Fix**: Generate favicons from the Omniprise logo (use a tool like https://realfavicongenerator.net) and place them in `public/`. Add the appropriate `<link>` tags in `layout.tsx`.

### 4.5 SEO files not committed to git

`robots.txt` and `sitemap.xml` exist locally but are untracked. If another developer clones the repo, these files are missing.

**Fix**: `git add public/robots.txt public/sitemap.xml` then commit.

## 5. Important Issues (Fix This Week)

### 5.1 No Open Graph image

`layout.tsx` defines OG metadata but has no `images` field:

```ts
openGraph: {
  title: "Omniprise — Plataforma de Marcas Gastronómicas",
  // Missing: images: ['/og-image.jpg']
}
```

**Impact**: When the URL is shared on WhatsApp, Facebook, LinkedIn, etc., no preview image appears.

**Fix**: Create a branded 1200x630 OG image, place it in `public/og-image.jpg`, and add:
```ts
images: [{ url: '/og-image.jpg', width: 1200, height: 630 }],
```

### 5.2 Image optimization disabled

`next.config.ts`:
```ts
images: { unoptimized: true }
```

**Impact**: Brand logos are served as raw PNGs with no WebP/AVIF conversion, no lazy loading, no responsive sizing.

**Fix**: Remove `unoptimized: true` and configure `remotePatterns` if needed. Vercel handles image optimization automatically.

### 5.3 Large brand images in PNG format

| Image | Current Size | Expected WebP |
|---|---|---|
| `Sammy's Express Pizza.png` | 660 KB | ~50-80 KB |
| `Barrio Pizzero.png` | 420 KB | ~40-60 KB |
| `PastaBox.png` | 424 KB | ~40-60 KB |
| `Rocco.png` | 148 KB | ~20-30 KB |
| Others | < 40 KB | Acceptable |

**Fix**: Convert to WebP using `sharp`, `cwebp`, or an online tool. Update references in `BrandsSection.tsx`.

### 5.4 `dangerouslySetInnerHTML` in WorkModal

**File**: `src/components/WorkModal.tsx` line 89
**Severity**: MEDIUM (code quality, potential XSS vector)

```tsx
<span dangerouslySetInnerHTML={{
  __html: step.text.replace(
    /Tu nombre completo \+ cargo al que aplicás/,
    '<strong class="text-text-primary font-medium">Tu nombre completo + cargo al que aplicás</strong>'
  )
}} />
```

While the content is hardcoded and not exploitable today, this is a bad pattern.

**Fix**: Use React elements instead:
```tsx
<span>
  <strong className="text-text-primary font-medium">
    Tu nombre completo + cargo al que aplicás
  </strong>
</span>
```
Or restructure the STEPS data to separate the bold portion.

### 5.5 Counter hydration flash

The animated stat counters show "0" during SSR and before the IntersectionObserver triggers. The live site text extraction confirms this:
```
0 Marcas en portafolio
0 Locales físicos activos
0 Colaboradores
```

**Fix**: Either render the final values server-side (no animation) or hide the section until client hydration completes.

### 5.6 `old-static/` directory still in git

80 KB of dead v1 code (old HTML site) still tracked in the repository.

**Fix**:
```bash
git rm -r old-static/
```
It's already in `tsconfig.json` exclude list and likely `.gitignore`, but the files are still tracked.

### 5.7 No analytics

No Vercel Analytics, Google Analytics, or any visitor tracking is configured.

**Fix**: Add `@vercel/analytics` (free tier) with one line in `layout.tsx`.

## 6. Minor Issues (Backlog)

### 6.1 No Twitter/X Card metadata

Missing `twitter:card`, `twitter:title`, `twitter:image` etc. in metadata. Links shared on X/Twitter will look bare.

### 6.2 No custom 404 page

Using Next.js default 404. Should have a branded `not-found.tsx`.

### 6.3 No `out/` cleanup

The `out/` directory (20 MB stale static export) is on disk. While `.gitignore`d, it can confuse developers.

### 6.4 No skip-to-content link

Accessibility: No way for keyboard users to skip past the navbar.

### 6.5 Scroll behavior uses `href="#"` for home

`Navbar.tsx` and `Footer.tsx` use `<a href="#">` for the logo link. Should be `<a href="/">` or use Next.js `<Link>`.

### 6.6 No `<Link>` component used

All internal navigation uses `<a>` tags instead of Next.js `<Link>`. This causes full page reloads on navigation instead of client-side transitions. For a single-page site this is minor, but if pages are added it will matter.

### 6.7 No `.env.example` or environment documentation

No `.env` files or `.env.example` found. If any environment variables are needed in the future, there's no documentation.
